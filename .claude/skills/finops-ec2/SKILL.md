# FinOps EC2 GPU Scheduling Skill

## Scope

> **Review-only authority:** LangGraph owns detection and calculations. Review checkpoint, SLA, warm-up, and residual-cost safety; do not rerun rule arithmetic.

Reduce EC2 accelerator instance-hours while preserving training jobs, inference
availability, checkpoint safety, and attached storage requirements.

## Input Mode

- **Orchestrator subagent mode** (`=== TERRAFORM ===` present): inline slices only.
- **Standalone mode**: scan `WORK_DIR` for full inputs.

## Detection Rules

Apply `rules/gpu_instance_unscheduled.json`.

| Rule | Condition | Severity | Action |
|------|-----------|----------|--------|
| EC2G1 | GPU/Inferentia/Trainium instance runs during low/off-hours with no schedule tag/control | HIGH | ADD_INSTANCE_SCHEDULER |
| EC2G2 | Dev/training accelerator has low utilization and no stop automation | HIGH | ADD_SSM_OR_EVENTBRIDGE_SCHEDULE |
| EC2G3 | Fixed GPU ASG desired capacity with low utilization and no scaling/schedule | MEDIUM | ADD_ASG_SCHEDULE_OR_SCALING |
| EC2G4 | Stopped-instance savings claimed but EBS/EIP residual cost ignored | INFO | INCLUDE_RESIDUAL_COSTS |

## Safety Guardrails

- Do not stop production inference instances without SLA, failover, and warm-up
  review.
- Confirm checkpointing and job-resume behavior before scheduling training hosts.
- Account for residual EBS, snapshot, Elastic IP, NAT, and data transfer costs.
- For ASGs, schedule desired/min/max capacity together and avoid fighting other
  autoscaling policies.

## Output

Write to `WORK_DIR/result/`:
- `finops_report.md` using `references/report.md`.
- `main_optimized.tf` only for safe scheduler tags, SSM associations, EventBridge
  schedules, or ASG scheduled actions. For AWS Solutions Instance Scheduler,
  output required tags and deployment prerequisites instead of embedding a full
  solution stack.
