# FinOps NAT Gateway Analysis Skill

## Scope

Reduce avoidable NAT hourly and per-GB data processing charges while preserving
private subnet egress, security policy, route correctness, availability, and
service access.

## Input Mode

- **Orchestrator subagent mode** (`=== TERRAFORM ===` present): inline slices only.
- **Standalone mode**: scan `WORK_DIR` for full inputs.

## Required Evidence

| File | Used For | If Missing |
|------|----------|------------|
| `main.tf` | NAT gateways, route tables, subnets, VPC endpoints, endpoint route assoc, policies, SGs, tags | Cannot analyze; ask for path |
| `metrics.json` | NAT bytes/packets, source subnet/AZ, destination service mix, VPC Flow Logs summary, endpoint traffic, route tables | Mark metrics unavailable |
| `cost_report.json` | Monthly NAT, VPC endpoint, data processing, data transfer cost | Mark cost unavailable |

Missing facts → write `Not available in the provided data; verify in the real environment.`

## Detection Rules (apply `rules/s3_nat_bypass.json`)

| Rule | Condition | Severity | Action |
|------|-----------|----------|--------|
| N1 | Same-region S3 traffic goes through NAT; no correct S3 gateway endpoint route | HIGH | ADD_S3_GATEWAY_ENDPOINT |
| N2 | Same-region DynamoDB traffic via NAT; no correct DynamoDB gateway endpoint route | HIGH | ADD_DYNAMODB_GATEWAY_ENDPOINT |
| N3 | High-volume AWS service traffic via NAT may be cheaper through interface endpoint | MEDIUM | MODEL_INTERFACE_ENDPOINT |
| N4 | Cross-AZ private subnet traffic uses NAT in another AZ | MEDIUM | REVIEW_AZ_LOCAL_NAT_OR_ENDPOINTS |
| N5 | NAT hourly cost remains after offload with no internet egress evidence | LOW | REVIEW_NAT_REMOVAL |
| N6 | Endpoint exists but route association, region, DNS, policy, or SG is wrong | HIGH | FIX_ENDPOINT_CONFIGURATION |

## Safety Guardrails

- Do not remove NAT gateways just because endpoint candidates exist. Workloads
  may still need NAT for internet egress, third-party APIs, package
  repositories, OS updates, SaaS endpoints, or services without VPC endpoint support.
- Gateway endpoint (free) → Interface endpoint → right-sizing — pick the cheapest valid path.

Full pre-change checklists: `references/details.md` § Required Safety Checks.

Write to `WORK_DIR/result/`:

- `main_optimized.tf` — preserve real names; add gateway endpoints based on real
  route tables; add interface endpoints only when cost modeling supports them.
  Full rules: `references/details.md` § Optimized Terraform.
