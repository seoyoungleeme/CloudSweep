# FinOps S3 Analysis Skill

## Scope

> **Review-only authority:** LangGraph owns detection and calculations. Review retention, Object Lock, replication, restore, and compliance safety; do not rerun rule arithmetic.

Reduce noncurrent version accumulation and stale storage while preserving
restore windows, legal hold, Object Lock, replication, backup, audit, and
compliance requirements.

## Detection Rules (apply `rules/missing_lifecycle_policy.json`)

| Rule | Condition | Severity | Action |
|------|-----------|----------|--------|
| V1 | Versioning enabled; noncurrent versions growing; no noncurrent expiration | HIGH | ADD_SAFE_LIFECYCLE_POLICY |
| V2 | Existing lifecycle lacks `abort_incomplete_multipart_upload`; incomplete uploads accumulate | MEDIUM | ADD_MULTIPART_ABORT |
| V3 | Object Lock, legal hold, compliance, replication, or backup evidence exists | INFO | DO_NOT_EXPIRE_WITHOUT_POLICY_REVIEW |
| V4 | Storage class mix suggests transition opportunity | LOW | MODEL_STORAGE_CLASS_TRANSITION |
| V5 | Missing owner, data class, retention, or cost tags | MEDIUM | ADD_GOVERNANCE_TAGS |
| R1 | GET/read request rate is high relative to upstream invocations; no cache evidence | HIGH | REDUCE_REQUESTS_OR_ADD_CACHE |

## Safety Guardrails

- Do not add aggressive expiration from version growth alone.
  `NoncurrentVersionExpiration` is permanent. Validate restore, compliance,
  Object Lock, replication, backup, and application rollback requirements first.
- Do not count savings when Object Lock or retention policy blocks expiration.
- Do not collapse request-cost findings into lifecycle recommendations. If S3
  cost is driven by GET/HEAD/LIST requests, attribute the caller first using
  RELATED_CONTEXT and recommend caching/request reduction. If caller or
  invocation data is missing, mark the request-amplification evidence gap.

Lifecycle baselines per environment: `references/details.md` § Lifecycle Baselines.
