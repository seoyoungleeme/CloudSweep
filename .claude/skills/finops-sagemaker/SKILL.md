# FinOps SageMaker Analysis Skill

## Scope

> **Review-only authority:** LangGraph owns detection and calculations. Review latency, model-load, GPU-memory, and minimum-capacity safety; do not rerun rule arithmetic.

Reduce SageMaker inference endpoint cost while preserving latency, availability,
model load time, and production safety.

## Input Mode

- **Orchestrator subagent mode** (`=== TERRAFORM ===` present): inline slices only.
- **Standalone mode**: scan `WORK_DIR` for full inputs.

## Detection Rules

Apply `rules/missing_endpoint_autoscaling.json`.

| Rule | Condition | Severity | Action |
|------|-----------|----------|--------|
| SM1 | Endpoint variant has fixed `initial_instance_count` and no scaling target/policy | HIGH | ADD_TARGET_TRACKING |
| SM2 | Predictable low/off-hours traffic and no scheduled scaling | MEDIUM | ADD_SCHEDULED_SCALING |
| SM3 | GPU/accelerator instance has low utilization and safe latency/error metrics | HIGH | RIGHTSIZE_VARIANT |
| SM4 | Traffic is bursty or idle-heavy and real-time endpoint runs continuously | LOW | EVALUATE_ASYNC_SERVERLESS_BATCH |

## Safety Guardrails

- Do not reduce instance count or size from average traffic alone. Check p95/p99
  latency, model load time, errors, throttling, queueing, and business SLA.
- For GPU models, validate memory headroom, batch size, and cold-start behavior.
- If the endpoint must serve 24x7 low-latency traffic, scheduled scaling should
  reduce to the minimum safe capacity, not remove all production capacity.
- For true zero-idle operation, evaluate async inference, serverless inference,
  batch transform, or managed API options instead of assuming a real-time
  endpoint can be safely stopped.
