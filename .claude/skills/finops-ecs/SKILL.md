# FinOps ECS/Fargate Analysis Skill

## Scope

> **Review-only authority:** LangGraph owns detection and calculations. Review workload headroom and rollout safety; do not rerun rule arithmetic.

Detect Fargate task right-sizing opportunities (CPU/memory over-provisioning),
missing Auto Scaling on static `desired_count`, and stale platform versions.

## Input Mode

- **Orchestrator subagent mode** (`=== TERRAFORM ===` present): inline slices only.
- **Standalone mode**: scan `WORK_DIR` for full inputs.

## Required Evidence

| File | Used For | If Missing |
|------|----------|------------|
| `main.tf` | ECS service and task definition resources | Cannot continue — ask user |
| `metrics/metrics.json` or `metrics.json` | `cpu_percent`, `memory_percent` per service (30-day hourly) | Mark metrics unavailable |
| `cost_report.json` | Monthly ECS + Fargate spend | Mark cost unavailable |

## Detection Rules (apply `rules/fargate_rightsizing.json`)

| Rule | Condition (summary) | Severity | Action |
|------|--------------------|----------|--------|
| E1 | Fargate; cpu_avg < 20% AND cpu_p95 < 50% AND cpu_max < 80% | HIGH | RIGHTSIZE_CPU |
| E2 | Fargate; mem_avg < 20% AND mem_p95 < 50% AND mem_max < 90% | HIGH | RIGHTSIZE_MEMORY |
| E3 | Static `desired_count` with no `aws_appautoscaling_target` | MEDIUM | ADD_ECS_AUTOSCALING |
| E4 | `platform_version != "LATEST"` | LOW | UPDATE_PLATFORM_VERSION |

Full conditions, rightsize math, valid Fargate CPU/memory combinations:
`references/details.md`.

## Safety Guardrails

- Do NOT reduce CPU if `cpu_max_pct >= 80%`.
- Do NOT reduce memory if `memory_max_pct >= 90%` (OOM risk).
- Right-sized target = `p95_actual * 1.3` (30% headroom), rounded up to valid
  Fargate value; CPU must support target memory range.

## Output

Write to `WORK_DIR/result/`:
- `parsed_input.json`, `findings.json`, `finops_report.md`, `main_optimized.tf`.
- `main_optimized.tf`: preserve `terraform {}`/`provider {}`; reduce `cpu`/`memory`
  in affected task definitions; keep `aws_ecs_service` unchanged. Full file/format
  rules: `references/details.md` § Output Files.
- Report template: `references/report.md`.
