# FinOps CloudWatch Log Group Analysis Skill

## Scope

> **Review-only authority:** LangGraph owns detection and calculations. Review retention/compliance safety and explain machine findings; do not rerun rule arithmetic.

Reduce CloudWatch Logs storage cost while preserving operational, security,
audit, and compliance requirements.

## Detection Rules (apply `rules/missing_retention_policy.json`)

| Rule | Condition | Severity | Action |
|------|-----------|----------|--------|
| C1 | `retention_in_days` missing or `== 0` | HIGH | SET_RETENTION |
| C2 | `retention_in_days > environment_max` without documented exception | MEDIUM | REDUCE_RETENTION |
| C3 | No ingestion and no recent events for full observation window | LOW | REVIEW_DELETE_CANDIDATE |
| C4 | Invalid or scenario-only Terraform retention attributes | MEDIUM | NORMALIZE_TERRAFORM |

## Safety Guardrails

- Valid Terraform attribute is `retention_in_days`; treat `retention_days`
  scenario data as evidence and normalize on output.
- Missing or `0` retention = indefinite. Default 90 days for unknown environments.
- Do not reduce audit / security / compliance retention from cost evidence
  alone — flag for owner validation.

Environment baselines and inference rules: `references/details.md` § Retention Baselines.
