## Step 2 — Run Pipeline Scripts

```bash

  --tf      <main.tf path> \
  --metrics <metrics.json path> \
  --cost    <cost_report.json path> \
  --out     WORK_DIR/parsed_input.json

# 2. Analyze
python SKILL_DIR/scripts/analyzer.py \
  --input WORK_DIR/parsed_input.json \

# 3. Format
```

## Step 3 — Deep Architectural Analysis

### 분석 원칙 (반드시 준수)

- **교차 증거 원칙**: 결론은 반드시 발견된 파일들의 교차 증거에만 기반할 것.
- **불확실성 명시 원칙**: 제공된 파일에 없는 정보는 **"제공된 데이터에서 확인 불가 — 실제 환경 검증 필요"** 로 표시한다.
- **범위 제한 원칙**: 파일에서 확인된 것 이상으로 서술하지 않는다.

Use the Read tool to read `main.tf` and `WORK_DIR/findings.json`. Analyze:

**3.1 Evidence from Infrastructure (Terraform)**
- 총 `aws_ebs_snapshot` 수, 고아 스냅샷 수
- `SourceVolumeStatus = "deleted"` 태그 패턴 확인
- 스냅샷 설명 필드에서 추가 단서 확인

**3.2 Evidence from Metrics (30 days)**
- 고아 스냅샷별 `storage_gb_avg` 요약 (대량일 경우 상위 5개 샘플 + 합계)
- 총 낭비 스토리지 GB

**3.3 Evidence from Cost Report (6 months)**
- 월별 EBS 비용 추세
- `pricing_note` 활용

**3.4 Root Cause**
- 볼륨 삭제 후 스냅샷 미정리 원인 (Lifecycle 정책 부재, 수동 삭제 누락 등)

**Proposed Solution**
- Immediate Actions: AWS CLI 대량 삭제 명령
- Preventive Actions: Data Lifecycle Manager, Lambda 자동화, 태깅 정책

**Optimized Terraform 생성 규칙**:
- 고아 스냅샷이 대량(10개 초과)인 경우: `main_optimized.tf`에 `terraform {}` + `provider {}` 블록만 남기고,
  나머지 `aws_ebs_snapshot` 리소스를 일괄 삭제 대상으로 명시한 주석 블록을 포함한다.
  개별 리소스를 하나씩 나열하지 말고, 대신 Terraform state 일괄 제거 명령과
  AWS CLI 대량 삭제 스크립트를 주석으로 제공한다.
- 고아가 아닌 스냅샷이 있으면 그 리소스는 원본 그대로 유지한다.
- 완성된 코드는 Write 툴로 `WORK_DIR/main_optimized.tf`로 저장하고 리포트에도 포함한다.
