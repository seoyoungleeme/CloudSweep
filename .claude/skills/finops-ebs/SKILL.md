# FinOps EBS Snapshot Analysis Skill

## Scope

Reduce unnecessary snapshot storage, archive eligible long-term backups, and
improve lifecycle governance without deleting recovery points still required
for AMIs, launch templates, AWS Backup, DR, audit, legal hold, or compliance.

## Input Mode

- **Orchestrator subagent mode** (`=== TERRAFORM ===` present): inline slices only.
- **Standalone mode**: scan `WORK_DIR` for full inputs.

## Required Evidence

| File | Used For | If Missing |
|------|----------|------------|
| `main.tf` | `aws_ebs_snapshot` and related lifecycle resources | Cannot analyze; ask for path |
| `metrics.json` | Snapshot storage size, age, archive tier, FSR, restore/access evidence | Mark metrics unavailable |
| `cost_report.json` | Monthly EBS snapshot cost, pricing notes | Mark cost unavailable |

Missing facts → write `Not available in the provided data; verify in the real environment.`

## Detection Rules (apply `rules/orphaned_snapshot.json`)

| Rule | Condition | Severity | Action |
|------|-----------|----------|--------|
| S1 | Source volume deleted and no dependency evidence | HIGH | REVIEW_DELETE |
| S2 | Snapshot referenced by AMI, launch template, AWS Backup, DLM, or legal/compliance tag | INFO | DO_NOT_DELETE |
| S3 | Older than archive threshold and retained for long-term backup | MEDIUM | CONSIDER_ARCHIVE |
| S4 | Fast Snapshot Restore enabled without restore/launch need | MEDIUM | REVIEW_FSR_DISABLE |
| S5 | Lacks owner, retention, or purpose tags | MEDIUM | ADD_GOVERNANCE_TAGS |

## Safety Guardrails

- Do not recommend deletion based only on `SourceVolumeStatus = deleted` —
  snapshots may still be required by AMIs, launch templates, recovery runbooks,
  AWS Backup recovery points, DLM policies, legal holds, or retention policy.
- Do not count archive savings when restore/retrieval charges or minimum archive
  duration would erase the benefit.

Full deletion checklist: `references/details.md` § Required Deletion Checks.
