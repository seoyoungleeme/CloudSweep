# FinOps Bedrock Analysis Skill

## Scope

> **Review-only authority:** LangGraph owns detection and calculations. Use this Skill to review safety, evidence gaps, MCP results, and explanation. Do not recalculate thresholds or savings.

Reduce Bedrock inference cost while preserving answer quality, latency, safety,
freshness, and model availability.

- **Orchestrator subagent mode** (`=== TERRAFORM ===` present): inline slices only.
- **Standalone mode**: scan `WORK_DIR` for full inputs.

## Detection Rules

| Rule | Condition | Severity | Action |
|------|-----------|----------|--------|
| B1 | On-Demand spend exceeds modeled PT/reserved capacity cost for steady traffic | HIGH | MODEL_THROUGHPUT_COMMIT |
| B2 | PT/MU or reserved capacity exists but utilization is low | HIGH | REVIEW_COMMITTED_THROUGHPUT |
| B3 | Repeated static prompt prefix/context and no prompt cache read evidence | HIGH | ENABLE_PROMPT_CACHING |
| B4 | High repeated or similar user queries and no semantic cache layer | MEDIUM | ADD_SEMANTIC_CACHE |
| B5 | LLM workload has enough volume to compare managed API vs hosted inference | LOW | RUN_LLM_TCO_MODEL |

Apply `rules/bedrock_throughput_breakeven.json`,
`rules/missing_prompt_cache.json`, and `rules/missing_semantic_cache.json`.

## Safety Guardrails

- Do not recommend Provisioned Throughput or reserved capacity from total spend
  alone. Require stable token rate, latency/SLA need, and model availability.
- Do not cache prompts that include user-specific secrets, short-lived context,
  or rapidly changing policy content without explicit TTL and invalidation.
- Do not recommend semantic cache for regulated, personalized, or high-risk
  answers unless the report includes similarity threshold, freshness, and audit
  controls.
- Separate Bedrock prompt caching from semantic caching: prompt caching reuses
  repeated prompt prefixes; semantic caching reuses answers for similar queries.
